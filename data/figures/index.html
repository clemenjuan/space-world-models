LeWM Encoding And Prediction Architecture

LeWM supplies latent transition predictions. CEM searches action sequences. Linear probes and mission weights score predicted futures without changing the world model.

Network Architecture View: Matrices And Neurons

This is the neural-network view of the LeWM transition model. It shows how a short history of spacecraft observations and commanded actions is converted into latent vectors, then passed through a conditional transformer to predict the next latent state.

Interactive component explorer

1. Observation encoder

The observation encoder turns each 25-feature EventSat observation into a compact latent vector. It does this independently for each timestep in the recent history window.

Observation history 3 recent rows
25 features each
->
VectorEncoder MLP Linear 25 -> 256
LayerNorm + SiLU
Linear 256 -> 192
->
Encoded latent history 3 recent rows
192 channels each
Why a latent? The latent vector is the model's internal representation of spacecraft state. It is smaller and easier for the predictor to propagate than raw telemetry.

2. Projector

The projector is a small MLP after the observation encoder. It maps encoder outputs into the latent space where the prediction loss and SIGReg regularization are applied.

Encoder output 192 channels
->
Projector MLP 192 -> 512 -> 192
->
z_t latent used by predictor
Why use it? This follows JEPA-style practice: keep the observation encoder and prediction space separable, and give the representation a learned space where latent prediction is well conditioned.

3. Action embedder

Actions are mode commands such as charging, communication, or payload operations. The action embedder converts a 7-way one-hot command into a 192-channel conditioning vector.

Action history 3 recent rows
7 mode columns
->
1x1 Conv mixes mode columns
->
MLP embedder 7 -> 768 -> 192
->
Action conditioning 3 recent rows
192 channels each
Why embed actions? The predictor needs to know not only where the spacecraft is, but also which mode is being commanded. Action embeddings condition the dynamics.

4. Why history length is 3

The current model uses the last three observation/action timesteps. This is a design choice in the current config, not a law of nature.

Reasoning. Three steps give the predictor short-term temporal context: recent mode, resource trend, and pipeline progress. It keeps computation small for embedded deployment. For the thesis, history length should be ablated against prediction error and MPC performance.
What to test. Compare history sizes such as 1, 3, 5, and 8. Longer history may help eclipse/contact/pipeline dynamics but costs memory and can overfit if data is limited.

5. Predictor transformer

The ARPredictor is the transition model. It receives the latent history plus the action-conditioning history, then predicts the next latent state. This is the learned world model used by MPC.

Latent history3 rows x 192 channels
Action conditioning3 rows x 192 channels
Conditional transformer stack: repeated 4 times
Position embeddingTells the model which timestep in the history each row belongs to.
AttentionMixes information across the recent timesteps. Current config uses 8 heads.
AdaLN gateUses the action vector to modulate how the latent state is updated.
Feed-forwardApplies the per-timestep nonlinear update: 192 -> 512 -> 192.
Residual connections. Each transformer block adds an update to the existing latent sequence, instead of replacing it completely. This stabilizes prediction over short rollouts.

Glossary and design context

Batch: how many examples are processed together for efficiency.
History: recent timesteps visible to the predictor. Current value: 3.
25 features: the EventSat observation vector.
7 modes: the discrete satellite action choices.
192 channels: the internal latent width in the current AUTOPS config.
z_hat: the model-predicted future latent state.
What the network learnsLatent transition dynamics: given recent state/action history, predict the next latent state.
What it does not learnIt is not directly a policy. CEM chooses actions outside the neural network.
Where probes enterAfter rollout, linear probes read predicted latents and convert them to mission attributes.
