# Step 1 — Orbit-Determination Environment wired to LeWM

**Date:** 2026-06-17
**Status:** Approved design (pending spec review)
**Scope:** One environment, one model, a complete training loop, and tests. No scheduling (Step 2), no FDIR (Step 3).

---

## 1. Goal

Adapt **LeWorldModel (LeWM)** — a JEPA-based world model — from pixel observations to a
satellite **orbit-determination (OD)** task, and run its full training loop end-to-end on
data generated by a custom Gymnasium environment.

LeWM recap:

```
Encoder:   z_t = enc(o_t)                  # observation -> 192-dim latent
Predictor: z_hat_{t+1} = pred(z_t, a_t)    # latent + action -> next latent
Loss:      L = L_pred + lambda * SIGReg(Z) # prediction MSE + Gaussian regularizer
```

The **predictor (`ARPredictor`)** and **`SIGReg`** are reused **verbatim** from the LeWM
baseline. Only the **encoder** (ViT → MLP) and the **`encode()` plumbing** (pixels → 4-dim
vector) change.

---

## 2. Baseline reference (le-wm)

Source: <https://github.com/lucas-maes/le-wm> (cloned & inspected). Key facts that constrain
this design:

- `module.py` contains `SIGReg`, `ARPredictor`, `Embedder` (action encoder), `MLP`,
  `Transformer`, `ConditionalBlock` — self-contained, depending only on `torch` + `einops`.
  **These do NOT live in `stable_pretraining`/`stable_worldmodel`** — they are local to the
  repo. So we base our repo on a copy of the le-wm code.
- `jepa.py`'s `JEPA.encode()` is hardcoded to pixels (`info['pixels']`, ViT
  `last_hidden_state[:, 0]`). This is the **one** method we override.
- Loss (`train.py:39-41`): `pred_loss = (pred_emb - tgt_emb).pow(2).mean()` plus
  `lambda * sigreg(emb.transpose(0, 1))`, with `lambda = 0.09`.
- Hyperparameters: `embed_dim = 192`, `history_size = 3`, `num_preds = 1`.
- Training harness: `hydra` configs + `stable_pretraining.Module` / `DataModule` /
  `Manager` + Lightning `Trainer`, optimizer AdamW (lr 5e-5, wd 1e-3),
  scheduler `LinearWarmupCosineAnnealingLR`.

### Decisions taken during brainstorming

1. **Dependencies:** full LeWM stack (`stable_pretraining 0.1.7`, `stable_worldmodel 0.1.1`,
   `lightning`, `hydra-core`) — confirmed pip-installable on this machine.
2. **Dynamics:** **Orekit** via `orekit_jpype 13.1.5.0` (Java 16 present; `JAVA_HOME` set).
   This reverses the original "no Orekit yet" note — intentional.
3. **Propagator:** **`EcksteinHechlerPropagator`** (analytical, J2–J6 zonal harmonics) so the
   orbit exhibits real **sun-synchronous nodal precession**.
4. **Training:** wire the **complete** Lightning training loop now, not just a forward pass.
5. **Ground station:** Munich (48.15°N, 11.58°E, 0.5 km). **Orbit:** SSO @ 400 km.

## 4. Environment — `envs/od_env.py`

A Gymnasium env wrapping Orekit for ground-truth dynamics and the measurement model.

### 4.1 Orekit setup
- One-time JVM init + Orekit data (orekit data zip) loaded at import.
- Inertial frame: `EME2000` (≈ ECI / J2000). μ, Re, J2…J6 from `Constants.EIGEN5C_*` /
  `Constants.WGS84_EARTH_MU` / `Constants.EIGEN5C_EARTH_C20` as required by
  `EcksteinHechlerPropagator`.
- Earth: WGS84 `OneAxisEllipsoid` in `ITRF`.

### 4.2 Hidden state
- 6-dim ECI `[r_x, r_y, r_z, v_x, v_y, v_z]` (meters, m/s) read from the propagated
  `SpacecraftState` PVCoordinates in `EME2000`.

### 4.3 Dynamics (ground truth)
- **`EcksteinHechlerPropagator`** built from the current orbit; J2–J6 zonal model →
  sun-synchronous nodal precession.
- **dt = 30 s** per step. `step()` propagates from `t` to `t + 30 s`.
- **Action** `a_t = [a_x, a_y, a_z]` in LVLH: plumbed end-to-end but **zero** in Step 1
  (coasting). Dynamics ignore it (documented). Continuous thrust → `NumericalPropagator`
  deferred to a later step.

### 4.4 Initial orbit (`reset`)
- SSO @ 400 km: `a = 6778.137e3 m`, `e ≈ 1e-3`, `i = 97.0°` (sun-synchronous for ~400 km),
  RAAN/argument-of-perigee/true-anomaly sampled per seed within configurable ranges.
- Seeded via Gymnasium `reset(seed=...)`.

### 4.5 Measurement model
- Configurable ground station (default Munich 48.15°N / 11.58°E / 0.5 km) as a
  `GeodeticPoint` on the WGS84 ellipsoid → `TopocentricFrame`.
- Orekit computes **range**, **azimuth**, **elevation** directly; **range-rate** from the
  relative PV of satellite w.r.t. the station frame. **ECI↔ECEF rotation handled by Orekit
  transforms** — no hand-rolled GMST.
- **Obs** `o_t = [range, azimuth, elevation, range_rate]` (m, rad, rad, m/s) + per-component
  i.i.d. Gaussian noise. Defaults: range σ = 10 m, angles σ = 0.01 rad, range-rate σ = 0.1 m/s.
  All configurable via constructor.
- **No visibility masking** in Step 1 (negative elevation allowed; documented). Obs always
  well-defined.

### 4.6 Gym spaces
- `observation_space = Box(shape=(4,))` in raw physical units (generous low/high bounds).
- `action_space = Box(low=-1, high=1, shape=(3,))`.
- Normalization is the **model's** responsibility (LeWM column normalizers), not the env's.

### 4.7 `info`
- `step()`/`reset()` return `info` containing the true 6-dim ECI state and current orbital
  elements (for tests and later evaluation).

## 5. Model — encoder swap, predictor/SIGReg unchanged

### 5.1 `models/od_encoder.py` — `OdEncoder`
- MLP `4 → 256 → 192`, SiLU activations, LayerNorm — same idiom as `module.MLP`/`Embedder`.
- Consumes obs `(B, T, 4)`, applied per timestep, outputs `(B, T, 192)`.

### 5.2 `models/od_jepa.py` — `ODJEPA(JEPA)`
- Subclasses baseline `JEPA`. Overrides **only** `encode(info)` to read `info["obs"]`
  `(B, T, 4)`, run `OdEncoder` per timestep, then the existing `projector`; sets
  `info["emb"]` and `info["act_emb"] = action_encoder(info["action"])`.
- `predict()`, `ARPredictor`, `Embedder` action encoder, `pred_proj`, and **`SIGReg`** used
  **as-is** from the baseline.
- `embed_dim = 192`, `history_size = 3`.

### 5.3 Loss
- `od_lejepa_forward` — math identical to baseline `lejepa_forward`:
  `pred_loss = (pred_emb - tgt_emb).pow(2).mean()`,
  `sigreg_loss = sigreg(emb.transpose(0, 1))`,
  `loss = pred_loss + 0.09 * sigreg_loss`. Reads `obs` not `pixels`.

## 6. Complete training loop

Mirrors `train.py` with OD substitutions.

### 6.1 `data/generate_dataset.py`
- Roll out `N` episodes of `OdEnv` (zero action), each length `L` steps.
- Save trajectories of `obs (L, 4)` + `action (L, 3)` (+ true state for reference) to disk
  (`.npz` or HF-dataset on disk).

### 6.2 `od_datasets/od_dataset.py`
- Windowed `torch.utils.data.Dataset`: yields `{"obs": (T, 4), "action": (T, 3)}` with
  `T = history_size + num_preds` windows sliced from trajectories.
- `stable_pretraining.data.DataModule` + `DataLoader` (train/val split, seeded).

### 6.3 Normalizers
- `get_column_normalizer` (baseline `utils.py`) on the 4 obs columns and 3 action columns,
  replacing the baseline image preprocessing.

### 6.4 `train_od.py`
- Hydra entry point reading `config/train/od.yaml`.
- `hydra.utils.instantiate(cfg.model)` → `ODJEPA`.
- `spt.Module(model=..., sigreg=SIGReg(**kwargs), forward=partial(od_lejepa_forward, cfg=cfg),
  optim=...)`.
- Optimizer AdamW (lr 5e-5, wd 1e-3), scheduler `LinearWarmupCosineAnnealingLR`.
- `pl.Trainer(accelerator="auto", precision=32, max_epochs=<modest>, gradient_clip_val=1.0)`
  — **CPU-safe** (no GPU/bf16 assumption).
- Checkpoint via `spt.Manager`.
- **W&B logging:** reuse the baseline `WandbLogger` path (gated by `cfg.wandb.enabled`,
  default `true`). Entity `sps-tum`; project read from `WANDB_PROJECT` env var with a
  default of `space-world-models` (rename once the W&B project is created). Hyperparameters
  logged via `logger.log_hyperparams`. Disable cleanly (`wandb.enabled=false`) for offline
  CI / the train-smoke test.

### 6.5 Configs
- `config/train/od.yaml`: `embed_dim=192`, `history_size=3`, `num_preds=1`,
  `loss.sigreg.weight=0.09`, `loss.sigreg.kwargs={knots:17,num_proj:1024}`, CPU-safe trainer.
- `config/train/model/od.yaml`: `encoder=OdEncoder`, baseline `predictor=ARPredictor`,
  `action_encoder=Embedder(input_dim=3)`, `projector`/`pred_proj` MLPs as baseline.
- `config/train/data/od.yaml`: dataset path, window size, batch size, split.
- `wandb` block in `config/train/od.yaml`: `enabled: true`, `config.entity: sps-tum`,
  `config.project: ${oc.env:WANDB_PROJECT,space-world-models}`. The train-smoke test sets
  `wandb.enabled=false`.

## 7. Tests — `tests/test_smoke.py`

1. **Rollout/shape:** roll out **200 steps** (zero action); assert obs `(200, 4)`, state
   `(200, 6)`, all finite.
2. **Model shapes:** build `(1, T, 4)` obs + `(1, T, 3)` action → `ODJEPA.encode` → `emb`
   is `(1, T, 192)`; `predict` → `(1, T, 192)`; `SIGReg(emb.transpose(0,1))` → finite scalar.
3. **Bounded invariants (replaces machine-precision energy check, since J2 is present):**
   - **z-angular-momentum** `h_z = x·v_y − y·v_x` conserved under axisymmetric zonal gravity:
     assert relative variation over 200 steps below a tight tolerance.
   - **J2-inclusive specific energy** `E = v²/2 − U`, with
     `U = μ/r [1 − J2 (Re/r)² P₂(sinφ)]`, `P₂(x)=(3x²−1)/2`, φ = geocentric latitude:
     assert relative variation bounded (loose tolerance appropriate to an analytical
     mean-element propagator).
   - **SSO sanity:** RAAN precession rate ≈ **+0.9856°/day** within tolerance — directly
     validates the J2/sun-synchronous physics.
4. **Train smoke:** tiny generated dataset, `Trainer(fast_dev_run=True)` (or 1 epoch); assert
   the step runs end-to-end and `loss`/`pred_loss`/`sigreg_loss` are finite.

## 8. Out of scope (explicit)

- Scheduling / observation planning (Step 2).
- Fault detection, isolation & recovery — FDIR (Step 3).
- Continuous thrust dynamics (needs `NumericalPropagator`).
- Visibility masking, multi-station, atmospheric drag / SRP, measurement bias.
- Full hyperparameter-tuned training run (this loop is correctness-first, CPU-safe).
